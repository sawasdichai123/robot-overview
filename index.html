Only Error — C24 "อะไรก็ได้"
LLM classified as ASK (ambiguous target) instead of REJECT (invalid command). The phrase "anything will do" was interpreted as under-specified rather than meaningless — a philosophically reasonable failure mode worth discussing in the paper as an LLM boundary case.
ASK Clarification Quality
Position-based clarification (ซ้าย/ขวา, left/right) resolved 5 / 10 ASK cases to EXECUTE. Color-based clarification failed on C12/C17/C18 — YOLO detects clear bottles as "white". C11/C15/C20 hit NO_RESPONSE timeout — TTS_PLAY_DELAY 4 s too short for Thai TTS length.

Key Finding — Tuning > Parameters
All four models share the same ≤7B size class, yet accuracy spans 0 – 96.7%. Qwen2.5's Thai multilingual instruction-tuning explains the gap over Mistral and InternLM2. Architecture and parameter count alone cannot predict performance for bilingual structured output.

Implication for the Paper
The offline benchmark (Section V-C) is a reliable proxy for full-pipeline accuracy. The single error in the live experiment (C24 "อะไรก็ได้" → ASK) is the same error pattern seen in the benchmark (REJECT category 90%), confirming that the LLM's boundary between "ambiguous" and "invalid" is consistent across evaluation modes. End-to-end decision latency (avg 11.3 s) is dominated by LLM inference (≈9–10 s on Mac M3 via Ollama) plus ROS pub/sub overhead — within acceptable range for lab pick-and-place tasks.
